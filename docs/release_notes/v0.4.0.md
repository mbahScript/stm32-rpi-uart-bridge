# v0.4.0 – Real TfL API Integration

## Summary

This release transitions the project from mock telemetry to live external API integration.

---

## New Features

- Integrated TfL Unified API
- Compact data formatting on Raspberry Pi
- New command: `TFL=<summary>`
- STM32 storage of latest summary
- New command: `GETTFL`
- Reliable SEQ-safe retry handling

---

## Technical Highlights

- Full bidirectional UART protocol
- Duplicate detection
- Idempotent retries
- Cross-platform architecture (Linux ↔ MCU)

---

## Example

### Send TfL Data
CMD|HOST|226|TFL=TUBE: Bakerloo=Good Service; ...|CHK


### STM32 Response


ACK|BUS01|226|TFL_RX|CHK


### Retrieve Stored Summary


CMD|HOST|164|GETTFL|CHK


### STM32 Returns


STATUS|BUS01|164|TFL=TUBE: Bakerloo=Good Service; ...|CHK
